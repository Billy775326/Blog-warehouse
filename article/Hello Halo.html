确保 x11vnc.service 和 napcat.service 都配置正确后，可以检查并修复任何可能的错误。首先，我们需要确保 x11vnc.service 正常运行，并且 napcat.service 可以依赖于它。

  1. 检查 x11vnc.service 文件：确保其配置正确，并且能够正常启动。

    ini

    复制代码

    [Unit] Description=Start x11vnc at startup. After=multi-user.target [Service] Type=simple ExecStart=/usr/bin/x11vnc -auth guess -forever -noxdamage -repeat -rfbauth /etc/x11vnc.pass -rfbport 5900 -shared [Install] WantedBy=multi-user.target

  2. 检查 napcat.service 文件：

    ini

    复制代码

    [Unit] Description=NapCat Service After=network.target x11vnc.service Requires=x11vnc.service [Service] Type=simple ExecStart=/bin/bash /home/Bot/NapCat/napcat.sh -q 2772757923 WorkingDirectory=/home/Bot/NapCat Restart=always RestartSec=10 User=root #Group=root # 替换成执行命令的用户组 [Install] WantedBy=multi-user.target

  3. 重新加载 systemd 配置：

    bash

    复制代码

    sudo systemctl daemon-reload

  4. 启用和启动服务：

    bash

    复制代码

    sudo systemctl enable x11vnc.service sudo systemctl start x11vnc.service sudo systemctl enable napcat.service sudo systemctl start napcat.service

  5. 检查服务状态：

    bash

    复制代码

    sudo systemctl status x11vnc.service sudo systemctl status napcat.service
